# 3가지 RAG기반 비정형 데이터 분석 모델을 모듈화해서 하나의 워커 에이전트로 만들기
ESG 보고서 분석 모델, NEWS 분석 모델, 증권사 리포트 분석 모델을 각각 모듈화하여 하나의 워커 에이전트로 만들고자해. 
목적: 하나의 워커 에이전트로 만들어서 매니저 에이전트가 종목/포트폴리오 추천 이유와 리스크, 기업/산업 분석을 자연어로 사용자에게 제공할 때 활용될거야.

- crewai, chromadb, fastapi 사용중

## ESG 보고서 분석 모델 (esg_model)
- 파일:
    - analyzer.py
    - blueprint.py
    - __init__.py


## NEWS 분석 모델(news_model)
- 파일:
    - app_news_worker.py
    - pipeline_news_analysis_mvp.py
    - crew_news_worker.py
    - ./news_chroma_db

- 데이터가 없을 시에는 90일치 최신 뉴스 데이터를 가져오고, 업데이트 되어있지 않은 최신 데이터만 가져오기. 90일이 지난 데이터는 자동 삭제되도록하기. 최신 데이터 업데이트는 매일 오전 8시로 할 것.

## 증권사 리포트 분석 모델 (reports_model)
- 파일:
    - 01_naver_report_crawler.py
    - 02_report_information_extractor.py
    - 03_chroma_db_loader.py
    - 04_rag_agent_query_test.py
    - reports_main.py
- DB 폴더:
    - reports
    - reports_chroma_db
    - parsed_data

- DB는 .env에 입력해둔 mysql 기반 데이터 베이스와 연동해줄 것.
